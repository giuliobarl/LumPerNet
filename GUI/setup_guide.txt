To run the PL/EL/JV measurement application, you can either click on the
'EL/PL ONE' icon below, or you can open the terminal
and enter the following command:

python3 /home/chose/Documenti/Scientific_Camera_Interfaces/SDK/Python_Toolkit/EL_PL_one.py

This is preferable, as the terminal itself will act as a debugging console,
printing the progress of the tests.

Before opening the application, PLEASE CAREFULLY READ THE README TEXT REPORTED
HERE:


README
This is the user guide to this GUI, developed to have easy access to the degradation imaging bench.
Before running the tests, make sure that:
- the camera USB cable is plugged in
- the power source is on at 24—26 V and the current knob is turned all the way to the max
    (during operation, the current should be around 1.8 A)
- the Arkeo PC is ON, connected to the network (IP address should be 192.168.0.216),
    and `Arkeo Multichannel` is running.

This app integrates PL/EL imaging and JV characterization. Before starting, go through the Tabs in
    the Box below, click on the buttons, and enter the desired values.
    - LIGHTS: Enter the time intervals over which the LEDs are on/off. Make sure that the White LED
        stays on for at least the duration of a JV measurement.
    - CAMERA: Enter the exposure time in milliseconds (e.g. entering 200 will result in an exposure time
        of 0.2 seconds). Please notice: if you enter 0 as the exposure time, the program will still run,
        but no image will be captured.
    - CYCLE: Enter the desired time interval between consecutive when measurements and acquisition strategy
        for cyclic mode. Acquisition supports either a 'linear' or a 'decreasing' strategy:
        - 'linear': performs one measurement at each iteration;
        - 'decreasing': performs measurements more frequently in the early iterations, less in later ones.
    - BATCH: Enter the number of active channels, the desired voltage for EL biasing, the cell area,
        the reference of the cell/batch, and the name of the researcher performing the measurements.
The acquired image(s) will be saved as '<reference>.tiff' in the folder
    '/home/chose/Documenti/<researcher>/<date>/<EL/PL_oc/PL_sc>'.

The command line at the bottom of the GUI allows running the measurement programs. It accepts 4 commands:
- summary: shows the values stored by the program, always check them before running.
- run: performs one single measurement and acquires an image.
- cycle: runs in cyclic mode, acquiring one image per iteration, until stopped.
    The sampling strategy is user-defined.
- stop: stops the running cycles. Always use this and only this command to stop cycles.

IMPORTANT: counterintuitively, the iterations following the first one start with the pause.
    This will be fixed in the future, but it is necessary as of now.
    This means that if you use the STOP command during the pause between iterations,
    you still need to wait for the JV, EL and PL measurements to take place.

If the program returns an error, and the LEDs remain ON, please CLOSE the application and read the
    README file on the desktop. You will find what to do in lines 41-45.
